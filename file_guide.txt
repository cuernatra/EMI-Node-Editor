====================================================================
1) ARKKITEHTUURI KARTALLA (MITÄ MISSÄKIN TAPAHTUU)
====================================================================

1.1 App + käynnistys
--------------------
src/main.cpp
  - Sovelluksen entrypoint.

src/app/app.h + src/app/app.cpp
  - App-luokka: init, update-loop, elinkaari.

src/app/ui.h + src/app/ui.cpp
  - Paneelien koordinointi, editorin pää-UI-runko.

src/app/constants.h
  - Yhteiset UI-vakiot (mitat, spacing, fonttiin liittyvät valinnat).

src/CMakeLists.txt
  - Mitä src:stä buildataan ja millä riippuvuuksilla.


1.2 Core graph -data
--------------------
src/core/graph/types.h
  - NodeType + PinType. Tämä vaikuttaa käytännössä kaikkialle.

src/core/graph/pin.h
  - Pin-rakenne.

src/core/graph/link.h
  - Link-rakenne (pin -> pin).

src/core/graph/visualNode.h + visualNode.cpp
  - Node-instanssin data + osin piirtologiikka/editorikäytös.

src/core/graph/idGen.h
  - ID-generointi.


1.3 Registry + factory (mitä nodeja on olemassa)
-------------------------------------------------
src/core/registry/nodeDescriptor.h
  - Node-/Pin-/Field-descriptor-mallit.

src/core/registry/nodeRegistry.cpp
  - Varsinainen node-katalogi: nimet, pinit, fieldit, compile-callbackit.

src/core/registry/nodeFactory.cpp
  - Luo VisualNode-instanssit descriptorien perusteella.

src/core/registry/fieldWidget.cpp
  - Inspectorin field-widgetit (UI-käytettävyys).


1.4 Compiler polku (graph -> AST -> VM)
----------------------------------------
src/core/compiler/pinResolver.cpp
  - Selvittää datavirran lähteet input-pinneille.

src/core/compiler/graphCompiler.cpp
  - Muuntaa visual graphin AST:ksi.

src/editor/graphCompilation.cpp
  - Orkestroi compile/run-ketjun editorista VM:lle.


1.5 Editor (tila + canvas + serialisointi)
-------------------------------------------
src/editor/graphState.h + graphState.cpp
  - Keskitetty tila (nodes, links, compile-status, ym.).

src/editor/graphEditor.cpp
  - Pääcanvas ja interaktiot.

src/editor/graphEditorUtils.cpp
  - Apulogiikka: layout-sync, validointi, linkkien huolto, migration-tyyppiset fixit.

src/editor/graphSerializer.cpp
  - Save/load-formaatti.


1.6 UI-paneelit
---------------
src/ui/topPanel.cpp        - yläpalkki
src/ui/leftPanel.cpp       - node-lista
src/ui/dropBar.cpp         - drag & drop
src/ui/nodePreview.cpp     - noden esikatselu
src/ui/consolePanel.cpp    - logit
src/ui/graphPreviewPanel.cpp - erillinen render-preview


====================================================================
2) MUUTOSRESEPTIT (ASKEL ASKELEELTA)
====================================================================

2.1 Uusi node (suositusjärjestys)
---------------------------------
1) Lisää NodeType: src/core/graph/types.h
2) Lisää descriptor: src/core/registry/nodeRegistry.cpp
3) Varmista factory-yhteensopivuus: src/core/registry/nodeFactory.cpp
4) Päivitä editor-layout/sync: src/editor/graphEditorUtils.cpp
5) Lisää compile-tuki: src/core/compiler/graphCompiler.cpp
6) Lisää save/load tuki: src/editor/graphSerializer.cpp
7) Tarkista node-preview: src/ui/nodePreview.cpp


2.2 Kun vaihdat pinin tai fieldin nimeä
---------------------------------------
Pakolliset paikat:
- src/core/registry/nodeRegistry.cpp
- src/core/compiler/graphCompiler.cpp
- src/editor/graphEditorUtils.cpp
- src/editor/graphSerializer.cpp
- src/ui/nodePreview.cpp

Muistisääntö:
"Nimenmuutos yhdessä paikassa = bugi viidessä paikassa"


2.3 Compile-logiikan muutos
---------------------------
Koske ainakin näihin:
- src/core/compiler/graphCompiler.cpp
- src/core/compiler/pinResolver.cpp
- src/editor/graphCompilation.cpp

Tarkista lopuksi:
- kääntyykö projekti
- tuleeko runtimeen uusi virheviesti
- näkyykö muutos oikein Debug Printissä / previewssä


2.4 Preview-renderöinnin muutos
-------------------------------
Pääfile:
- src/ui/graphPreviewPanel.cpp

Lisäksi tarkista:
- src/app/ui.cpp (ikkunan avaaminen/ohjaus)
- src/editor/mainEditor.cpp (integraatio)


====================================================================
3) VIKATILANTEET -> MISTÄ ETSIN ENSIN?
====================================================================

"Node ei spawnaa oikein"
  nodeRegistry.cpp -> nodeFactory.cpp -> graphEditor.cpp

"Linkki on olemassa mutta arvo väärä"
  graphEditorUtils.cpp -> pinResolver.cpp -> graphCompiler.cpp

"Graph ei lataudu oikein"
  graphSerializer.cpp -> types.h -> nodeRegistry.cpp

"Compile-nappi failaa"
  graphCompilation.cpp -> graphCompiler.cpp -> console-logit

"Preview näyttää väärin"
  graphPreviewPanel.cpp -> node field/pin nimet registryssä


====================================================================
4) TÄRKEIMMÄT SÄÄNNÖT (JOTKA SÄÄSTÄVÄT AIKAA)
====================================================================

Sääntö 1:
Jos muutat NodeTypea tai pin/field-nimeä,
päivitä aina compiler + serializer + editor utils + preview.

Sääntö 2:
Jos editorissa kaikki näyttää hyvältä mutta runtime ei toimi,
vika on usein compile-polussa (graphCompiler/pinResolver/graphCompilation).

Sääntö 3:
Tee mieluummin pieniä, fokusoituja committeja kuin yksi iso "sekamuutos".

5) HYVÄ "DONE"-CHECKLIST ENNEN COMMITTIA
====================================================================

====================================================================
6) PIENI SANASTO
====================================================================

Node descriptor:
  Määrittää node-tyypin pinni- ja field-rakenteen.

Pin resolver:
  Selvittää mistä input saa arvonsa linkkien kautta.

AST:
  Välimuoto, johon graph käännetään ennen VM-ajoa.

Flow node:
  Node, joka ohjaa suoritusjärjestystä (ei vain datalaskentaa).
