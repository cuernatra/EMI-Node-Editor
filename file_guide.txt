1) Entrypoint ja sovellus

==========================

src/CMakeLists.txt
Rooli: src-kansion build-määrittely
Vaikutus: jos lisäät tai poistat tiedostoja/kirjastoja, build rikkoutuu täällä ensimmäisenä

src/main.cpp
Rooli: ohjelman käynnistyspiste
Vaikutus: sovelluksen alustus, sammutus ja main loop
src/app/app.h

Rooli: App-luokan rajapinta
Vaikutus: jos muutat APIa → myös app.cpp ja käyttäjät päivitettävä
src/app/app.cpp

Rooli: sovelluksen runko (ikkuna, ImGui/SFML, loop)
Vaikutus: koko ohjelman toiminta (frame update, init-järjestys)
src/app/constants.h

Rooli: UI/layout/font-vakiot
Vaikutus: vaikuttaa lähes kaikkiin paneeleihin
src/app/ui.h

Rooli: UI:n pääkoordinaattori
Vaikutus: paneelien välinen kommunikointi
src/app/ui.cpp

Rooli: UI-looppi, splitterit, overlayt
Vaikutus: jos UI tuntuu sekavalta → katso tämä ensin


2) Core Graph – datamalli

==========================

types.h
NodeType + PinType enumit
Vaikutus: koko järjestelmä (editori, compiler, serializer, UI)
pin.h

Pin-rakenne (input/output)
Vaikutus: linkitys ja type check
link.h

Link-rakenne (pin → pin)
Vaikutus: editori, serializer, compiler
visualNode.h

Node-olio (id, title, pinni, kentät, sijainti)
Vaikutus: koko node-editorin perusyksikkö
visualNode.cpp

Nodejen piirtologiikka
Vaikutus: ulkoasu
idGen.h

Uniikkien ID:iden generointi
Vaikutus: jos menee rikki → editori + serializer bugaa


3) Registry ja Factory (node-määrittelyt)

==========================================

nodeDescriptor.h
Node-, Pin- ja Field-kuvaukset

nodeRegistry.cpp
Kaikki node-tyypit määritellään täällä
Vaikutus: uudet nodet, pinit, fieldit

nodeFactory.cpp
Luo VisualNode-instanssit descriptorista
Vaikutus: jos node spawn on rikki → vika usein täällä

fieldWidget.cpp
Inspectorin kenttäwidgetit
Vaikutus: editorin käyttökokemus


4) Compiler (graph → AST → runtime)

====================================

pinResolver.cpp
Selvittää mistä pinni saa arvonsa
Vaikutus: dataflow toimii oikein

graphCompiler.cpp
Koko käännöslogiikka
Vaikutus: ohjelman toiminta runtime:ssa

⚠️ Jos algoritmi (esim. A*) antaa väärää tulosta → tarkista tämä

Nopea debug-jäljitys

=====================

Node ei spawnaa oikein:
nodeRegistry.cpp → nodeFactory.cpp → graphEditor.cpp

Linkki toimii mutta arvo väärä:
graphEditorUtils.cpp → pinResolver.cpp → graphCompiler.cpp

Graph ei lataudu oikein:
graphSerializer.cpp + types.h + nodeRegistry.cpp

Compile-nappi antaa virheen:
graphCompilation.cpp → graphCompiler.cpp → console (EMI log)
